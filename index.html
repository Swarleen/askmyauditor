How This Agent Works
AskMyAuditor architecture — from natural language question to audit insight
person
Layer 1 · User Interface
You ask a question
Type any question in plain English on the AI Workspace tab — no SQL knowledge needed. "Which domains have the most overdue entities?" or "Show me all Critical entities with weak controls."
HTML · CSS · JavaScript Streamlit (live version)
psychology
Layer 2 · LLM Layer — Claude AI
Natural Language → SQL
The question is sent to Claude (Anthropic) along with the full database schema. Sophisticated prompt engineering converts your plain English question into a precise, safe SQL query.
-- Generated SQL
SELECT domain, COUNT(*) as overdue_count
FROM audit_universe
WHERE overdue_flag = 'Yes'
GROUP BY domain
ORDER BY overdue_count DESC;
check_circle SELECT only — no destructive queries ever executed
check_circle Exact column name matching from schema injection
check_circle Context-aware — understands audit domain terminology
check_circle Structured JSON output enforced for consistent rendering
database
Layer 3 · Data Layer
SQL Executed Against Real Data
The generated SQL runs against a real in-memory SQLite database loaded from the IT Audit Universe Excel file — 44 entities, 13 dimensions. Results are returned as a structured dataframe.

Layer 4 · Insight Layer
Results → Plain English Insight
Claude reads the query results and generates a 2-4 sentence plain English audit insight — explaining what the data means, the key finding, and a specific actionable recommendation. No technical jargon.

Answer: Cybersecurity leads with 8 overdue entities, followed by Cloud & Infrastructure with 7...
Key Finding: These two domains carry 52% of all coverage gaps despite being 43% of the universe.
Recommendation: Prioritise IAM and VAPT in the next audit cycle immediately.
